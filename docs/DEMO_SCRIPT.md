# 答辩演示稿

## 1. 项目介绍

本项目名称为“都市圈智慧通勤出行服务系统”，是《软件工程导论》课程设计的初级网页原型。系统面向都市圈跨区、跨市、高频通勤场景，重点解决多方式交通信息分散、路线推荐维度单一、绿色出行反馈不足的问题。

## 2. 原型边界

本系统是课程级原型，不接入真实地图 API，不接入真实后端，不进行真实导航。站点、线路、路线方案、实时状态均由 JavaScript 模拟，用户数据、收藏、历史记录和绿色出行统计通过浏览器 localStorage 保存。

## 3. 用户端演示流程

1. 打开 `index.html`，介绍首页中的系统定位和城市轨道交通主题。
2. 展示系统概览，包括模拟站点、模拟线路、路线方案和当前身份。
3. 使用默认账号 `student / 123456` 登录。
4. 在路线规划区输入起点“北京联合大学”、终点“国贸”，选择“时间优先”。
5. 点击“生成推荐路线”，展示路线卡片，包括用时、换乘、费用、拥挤度、碳减排和综合评分。
6. 切换“绿色优先”，说明系统会提高碳排放指标权重。
7. 点击“收藏路线”，展示收藏列表更新。
8. 查看历史记录，说明用户每次查询都会形成历史记录。
9. 查看实时交通信息模拟表格，说明线路状态和拥挤度均为课程模拟数据。
10. 查看绿色出行统计，说明系统会根据路线里程和交通方式计算碳减排与积分。

## 4. 管理员端演示流程

1. 打开 `admin.html`。
2. 使用默认管理员账号 `admin / admin123` 登录。
3. 展示数据概览，包括注册用户数、模拟线路数、历史查询量和累计减排量。
4. 展示用户管理表格，说明当前阶段仅做查看，不做真实数据库管理。
5. 展示线路数据管理表格，说明线路状态来自 `js/data.js`。
6. 展示热门路线统计，说明统计数据来源于用户端查询历史。
7. 展示绿色出行统计，说明后台可以看到累计低碳出行记录。

## 5. 技术路线说明

系统采用 HTML5、CSS3、JavaScript 实现。页面结构由 `index.html` 和 `admin.html` 承载；全局样式写在 `css/style.css`；模拟数据写在 `js/data.js`；localStorage 操作封装在 `js/storage.js`；路线规划和推荐算法写在 `js/routePlanner.js`；用户端交互写在 `js/app.js`；管理员端交互写在 `js/admin.js`。

## 6. 总结

本原型贴合课程报告中的用户端和管理员端设计，能够完成基本路线规划、路线推荐、实时交通模拟、绿色出行统计、收藏历史和后台统计展示。它的重点不是工业级导航能力，而是体现软件工程中的需求分析、系统设计、数据设计、算法设计和测试设计过程。
