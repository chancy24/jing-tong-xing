# 开发顺序

## 第一阶段：项目结构与静态页面

1. 建立 HTML、CSS、JavaScript、assets、docs 目录。
2. 创建用户端页面：`index.html`、`plan.html`、`traffic.html`、`green.html`、`history.html`。
3. 创建管理员端页面：`admin.html`。
4. 完成统一导航栏、页面标题、功能区域和基础卡片布局。
5. 完成蓝绿色科技风视觉基调，体现城市轨道交通主题。

## 第二阶段：模拟数据与本地存储

1. 在 `data.js` 中整理位置、线路、站点、路线方案和实时交通状态。
2. 在 `storage.js` 中封装 `localStorage` 读写方法。
3. 准备默认用户、管理员账号和示例通勤路线。
4. 确保刷新页面后收藏、历史记录和绿色统计仍可保留。

## 第三阶段：用户端核心功能

1. 实现注册、登录、退出和游客模式。
2. 实现起点、终点、出发时间和偏好策略输入。
3. 实现候选路线筛选、评分、排序和结果展示。
4. 实现多模式组合展示，例如“步行 + 地铁 + 公交 + 骑行”。
5. 实现收藏路线和历史记录。

## 第四阶段：绿色出行与实时交通

1. 实现碳减排计算。
2. 实现绿色积分和等级展示。
3. 实现线路状态、拥挤度、运营时间和预计到站时间模拟。
4. 将绿色出行统计与用户历史记录关联。

## 第五阶段：管理员端

1. 实现用户数量、路线数量、历史查询量和累计减排量概览。
2. 实现用户列表展示。
3. 实现线路数据表格展示。
4. 实现热门路线统计。
5. 实现绿色出行统计。

## 第六阶段：课程答辩完善

1. 按报告测试用例检查主要功能。
2. 补充异常输入提示，例如起点为空、终点为空、未登录收藏。
3. 检查 Chrome 和 Edge 浏览器兼容性。
4. 准备演示脚本和页面截图。
5. 明确说明原型边界：无真实地图 API、无真实后端、交通数据为模拟数据。
