# 京通行：面向首都都市圈的智慧通勤 MaaS 服务平台

`smart-commute-system` 是一个纯前端课程设计原型。最终版将用户端重构为更像真实移动端 App / Web App 的五个 Tab：`首页`、`通勤`、`绿色`、`消息`、`我的`，后台端保留为独立数据看板。

## 项目背景

首都都市圈存在大量高频通勤需求：高校学生早八到校、跨区跨城上班族到岗、园区员工最后一公里接驳、企业班车和停车压力、政府对热门 OD 与接驳短板的治理需求。京通行聚焦“每天重复发生的通勤”，关注稳定到达、迟到风险、低碳激励和匿名聚合治理分析。

## 产品定位

京通行不是普通导航 App。普通导航解决“这一次怎么走”，京通行解决“每天怎么更稳、更准时、更低碳”。

产品价值分三层：

- 用户通勤服务：固定通勤、稳定性评分、迟到风险预警、绿色积分、通勤月报；
- 单位通勤管理：高校/园区/企业看板、接驳需求、班车优化、绿色排行；
- 政府通勤治理：热门 OD、通勤走廊、接驳短板、绿色出行政策评估。

## 目标用户

- 北京联合大学学生；
- 跨区跨城上班族；
- 园区企业员工；
- 高校、园区、企业管理者；
- 政府交通治理部门。

## 核心功能

- App 化用户端：底部 Tab Bar、首页今日通勤、通勤页规划、绿色页积分、消息页提醒、我的页个人中心；
- 固定通勤路线：保存路线名称、起终点、常用出发时间、目标到达时间、身份类型和偏好；
- 多模式路线规划：地铁、公交、步行、共享单车、校园接驳车、园区班车、公交短驳；
- 通勤稳定性评分：综合准点率、拥挤度、换乘、步行距离、天气影响、最后一公里方式、用时和费用；
- 迟到风险预警：输出低/中/高风险、建议提前出发时间和更稳定路线建议；
- 最后一公里接驳：展示高校、园区、软件园和城市副中心的接驳方案；
- 绿色积分账户：绿色次数、碳减排、积分、等级、等效种树、兑换和排行榜；
- 通勤月报：本月次数、总时长、平均时长、常用路线、碳减排、积分、最稳定/最拥挤路线和优化建议；
- 消息提醒：迟到风险、线路拥挤、接驳短板、积分到账、月报和系统公告；
- 后台三类管理端：高校/园区、企业、政府治理；
- OD 分析：热门 OD Top 5、热门通勤走廊、接驳短板和治理建议；
- 数据来源与隐私合规说明；
- 演示模式：一键生成答辩演示数据。

## 技术栈

- HTML5
- CSS3
- JavaScript
- JSON 模拟数据
- localStorage 本地存储

不接入真实后端、不接入真实地图 API，不需要安装第三方依赖。

## 运行方式

方式一：直接打开文件。

1. 双击 `index.html` 打开用户端；
2. 双击 `admin.html` 打开后台；
3. 双击 `data-policy.html` 查看数据合规说明；
4. 双击 `operation.html` 查看运营策略；
5. 双击 `about.html` 查看产品定位说明。

方式二：使用 VS Code Live Server。

1. 使用 VS Code 打开 `smart-commute-system` 文件夹；
2. 安装 Live Server 插件；
3. 右键 `index.html`，选择 `Open with Live Server`；
4. 后台可在同一地址下打开 `admin.html`，数据合规页打开 `data-policy.html`。

方式三：使用本地静态服务器。

```bash
python -m http.server 5500
```

然后访问：

- 用户端：`http://127.0.0.1:5500/index.html`
- 后台：`http://127.0.0.1:5500/admin.html`

默认账号：

- 普通用户：`student / 123456`
- 管理员：`admin / admin123`
- 演示用户：点击“加载演示数据”自动生成 `张成学 / 123456`

## 演示模式使用方法

用户端：

1. 打开 `index.html`；
2. 在首页点击“演示模式”，或进入“我的”页点击“加载演示数据”；
3. 系统自动生成用户“张成学”、身份“北京联合大学学生”；
4. 自动生成固定通勤“霍营 → 北京联合大学”、目标到达 `08:00`、偏好“稳定优先”；
5. 自动生成 10 条历史查询、3 条收藏路线、绿色积分、通勤月报、消息提醒和后台 OD 模拟数据；
6. 在“我的”页点击“清除演示数据”可移除演示数据。

后台：

1. 打开 `admin.html`；
2. 使用 `admin / admin123` 登录；
3. 点击“加载演示数据”；
4. 查看数据概览、三类管理端、OD 分析、热门通勤走廊、接驳短板和绿色统计。

## 课程原型说明

当前版本仅用于课程设计和答辩展示：

- 路线、准点率、拥挤度、天气影响、碳排放、积分、OD 和后台统计均为模拟数据；
- 不采集真实定位；
- 不上传个人轨迹；
- 不接入真实公交、地铁、共享单车、地图或政府开放数据接口；
- 用户、固定通勤、收藏、历史、绿色积分、消息和演示数据均保存在浏览器 localStorage。
